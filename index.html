用每 5 秒 reload 的方式做 Polling

想像一下當網站變大的時候會怎麼樣

  • 假設平均同時有 10,000 個使用者在線上
  • 每 5 秒鐘重新整理一次網頁
  • 1 小時就會有 7,200,000 次 request
  • Server 就要 check session + Query DB 同樣次

開始一個線上遊戲專案

遊戲中的通知

  • Real-time 技術中最基礎的功能 Simple Messaging
  • 一對一單向資料傳輸
  • 可以用來實作需要即時收到資料、通知的功能
  • Cloud Service
    • hydna
    • Superfeedr
  • Self Hosted
    • SockJS

聊天室

  • Real-time 技術中最常見的功能 Pub/Sub
  • 訂閱特定 Channel( or Topic or Subject)
  • Cloud Service
    • PubNub
    • Pusher
    • Realtime.co
  • Self Hosted
    • Faye
    • Socket.io + redis

進入副本

  • Real-time 版的 API Call
  • 可以做更複雜的互動
  • Self Hosted
    • SignalR
    • socket.io

團隊競賽

  • 資料同步
  • 當資料改變時，就能收到資料變動的項目
  • Cloud Service
    • Firebase
    • METEOR
    • Realtime.co
    • simperium
  • Self Hosted
    • DERBY
    • Sails
